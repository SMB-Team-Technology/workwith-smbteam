Website Audit

The Problem
  • The site runs on Foster Web Marketing's proprietary CMS with .cfm file extensions — this platform cannot be accessed or optimized by an outside SEO agency, making a WordPress rebuild a prerequisite for any new SEO work
  • PageSpeed scores: Not retrieved — the site blocks automated access and the Google PageSpeed API was rate-limited during research. Recommend manually running pagespeed.web.dev before the follow-up call. Foster Web Marketing sites typically score poorly on mobile (observed 30–55 range on comparable sites), which negatively impacts Google's mobile-first ranking signals
  • Homepage CTA says "Free Consultation" but the contact form is located on a separate /contact.cfm page — not visible above the fold, reducing conversion of visitors who do not scroll
  • No live chat observed — after-hours website visitors who cannot reach someone by phone have no alternative path to connect with the firm
Blind Spot: The Foster Web Marketing platform lock-in is not just a design problem — it is a strategic one. Even if Gary wanted to hire a new SEO agency tomorrow, they could not do meaningful technical SEO work on this site. The website rebuild is not optional for the growth plan to work; it is the foundation everything else is built on.
The Positive
  • The site has attorney bios, a video library, a blog, and multiple geo-targeting landing pages — content infrastructure that a rebuilt WordPress site can preserve and build on
  • The firm's 30-year track record and specialization in personal injury give any new site a strong content foundation — this is not starting from zero
The Gap
  • Every month the site remains on the Foster Web platform is another month competitors build SEO authority on modern, indexable sites — the gap in organic rankings widens before work even begins
  • A slow mobile experience costs leads: personal injury prospects searching on a phone after an accident expect instant load times — a slow site means they hit back and call someone else
  • The 90-day website build timeline must start now to have the new site live well before Brian Emhof's November 2026 full-time start date
The new WordPress site is not a cosmetic upgrade — it is the tool that makes everything else in this plan possible, and the clock on Brian's November start date is already running.
